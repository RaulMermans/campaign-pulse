# Campaign Pulse Design Direction

Campaign Pulse should feel like an editorial strategy room for newsletter operators, not a generic admin dashboard.

## Visual System

- Soft neutral background with warm card surfaces.
- High-contrast ink typography for hierarchy.
- Clean cards, subtle borders, restrained shadows.
- Muted campaign accents drawn from the mock data.
- Saturation states should be clear but not loud.

## Interaction Principles

- Calendar and table sends open the same detail drawer.
- Intelligence should explain the month, not merely list metrics.
- Reports should read like strategist memos with evidence and next actions.
- Export/print behavior should stay simple and dependency-free.

## Constraints

- Keep the static JSON architecture for now.
- No backend, database, auth, upload/import, external APIs, AI calls, or PDF libraries unless explicitly requested.
- Read markdown first and optimize tokens during vibe-coding sessions.
